# Запуск модели для общения

Эта папка содержит код для запуска модели в терминале для интерактивного общения.

## Использование

### Базовый запуск (только базовая модель):

```bash
python chat.py --base_model "microsoft/DialoGPT-small"
```

### Запуск с дообученной LoRA моделью:

```bash
python chat.py \
    --base_model "microsoft/DialoGPT-small" \
    --lora_model "../fine_tuning/lora_model"
```

### С дополнительными параметрами:

```bash
python chat.py \
    --base_model "microsoft/DialoGPT-small" \
    --lora_model "../fine_tuning/lora_model" \
    --system_prompt "Ты полезный ассистент." \
    --max_length 512 \
    --temperature 0.7
```

### Запуск с поиском по датасету (RAG)

Если передать `--dataset_path`, чат будет доставать похожие примеры из датасета и подмешивать их в контекст. Это помогает снизить галлюцинации на вашей предметной области.

```bash
python chat.py \
    --base_model "ai-forever/rugpt3small_based_on_gpt2" \
    --lora_model "../fine_tuning/lora_rugpt3small" \
    --dataset_path "../example_dataset.json" \
    --retrieval_k 3
```

### Рекомендуемый `--system_prompt` для консультанта (меньше галлюцинаций)

Ниже — промпт, который заставляет ассистента задавать уточняющие вопросы и **не выдумывать факты** (контакты, адреса, точные цены/сроки, реквизиты).

```bash
python chat.py \
  --base_model "ai-forever/rugpt3small_based_on_gpt2" \
  --lora_model "../fine_tuning/lora_rugpt3small" \
  --dataset_path "../example_dataset.json" \
  --retrieval_k 5 \
  --system_prompt "Ты инженер-консультант по инженерным системам частных домов (отопление, вода, канализация, электрика, вентиляция). Твоя задача — объяснять простым языком и помогать выбрать решение.\
Правила:\
1) Если вопрос требует исходных данных — задай 2-5 уточняющих вопросов (площадь/этажность, регион, есть ли газ, мощность электричества, режим проживания, количество санузлов, пожелания по вентиляции/автоматике).\
2) Никогда не выдумывай факты: адреса, телефоны, e-mail, цены «в рублях» с точностью, сроки «день в день», реквизиты, названия конкретных объектов. Если данных нет — скажи 'Не знаю' / 'Нужно уточнить' и попроси вводные.\
3) Отвечай структурно: (а) короткий вывод, (б) уточняющие вопросы, (в) 1-2 варианта решения, (г) риски/ошибки, (д) следующий шаг.\
4) Если в контексте есть похожие примеры из датасета — опирайся на них и не противоречь им.\
Стиль: доброжелательно, уверенно, без воды."
```

Если хотите отвечать строго “как в датасете”, добавьте флаг:

```bash
python chat.py \
    --base_model "ai-forever/rugpt3small_based_on_gpt2" \
    --lora_model "../fine_tuning/lora_rugpt3small" \
    --dataset_path "../example_dataset.json" \
    --answer_from_dataset
```

## Параметры

- `--base_model`: Имя базовой модели с HuggingFace (обязательно)
- `--lora_model`: Путь к дообученной LoRA модели (опционально)
- `--system_prompt`: Системный промпт для модели (опционально)
- `--max_length`: Максимальная длина генерируемого ответа (по умолчанию: 512)
- `--temperature`: Температура генерации, контролирует случайность (по умолчанию: 0.7)
  - Более низкие значения (0.1-0.5) делают ответы более детерминированными
  - Более высокие значения (0.7-1.0) делают ответы более креативными

## Команды в чате

- `quit`, `exit` или `q` - выход из чата
- `clear` - очистка истории разговора

## Пример использования

```
Вы: Привет! Как дела?
Модель: Привет! У меня все отлично, спасибо! Как дела у тебя?

Вы: Расскажи анекдот
Модель: [модель генерирует анекдот]

Вы: quit
До свидания!
```

## Примечания

- Модель автоматически использует GPU если доступен
- История разговора ограничена последними 10 сообщениями для экономии памяти
- Если указан путь к LoRA модели, веса будут объединены с базовой моделью
